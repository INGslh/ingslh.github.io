3.1介绍Python编写GNURadio程序

这个教程一共有3个部分。第一个部分是如何去修改，创造或者简单理解GRC产生的Python文件。第二部分是如何从头开始创造我们自定义的面向对象的模块。最后我们通过一个实际的工程去得到更多的练习，同时关于在我们自己的工程中建造一个大体印象。和上一个教程一样，所有的内容包括图片，源程序和grc文件，包含在github界面,我们可以按照GRC教程中的指导下载到本地。
同样我们需要建两个文件夹，一个是“解决方案”文件夹，一个是“工程”文件夹：

/home/user/gnuradio/tutorials/solutions
/home/user/gnuradio/tutorials/work

一般来说，如果我们将鼠标悬停在GRC流程图上，我们能够看到相应的文件名。同样也应用于其他图片上。

3.2模块从哪里来？

现在我们已经介绍了一些修改GRC生成的Python文件方法。我们知道了如何给我们的模块添加函数以满足我们的项目需求。让我们来感受下这一功能。我们将制作一个模块，能够用一个我们指定的数字乘以一个信号。首先我们应该了解到GR自带一个用来创建面向对象的模块相对容易的工具——gr_modtool,让我们打开一个终端，从教程三的路径下开始

我们可以看到这里有很多指令可以使用，在本教程中我们只需要了解newmod和add；完整的说明能够在教程之外通过gr_modtool指令很容易的得到。

首先我们注意到除了在得到帮助之后的这些指令可以使用，我们也可以其他的指令得到更多的信息。让我们从newmod开始，因为它是创造一个out-of-tree模块的指令。

现在我们读完了newmod的指令集，可以推断出我们想要的是默认指令-n，我们可以在newmod之后输入我们想要的模块名字(MODULE_NAME)。实际上建议将-n避免作为其他命令，因为他会覆盖自动生成的名字。现在我们忽略其他的命令。

3.2.2 建立一个新模块

现在我们在当前目录下看到一个新的文件夹——gr_tutorial。让我们看下这个文件夹，看下gr_modtool为我们做了什么。

由于本教程针对Python，所以我们只需要关心Python文件夹和grc文件夹。在深入研究代码之前，我们需要从模板中创建一个组件（block），实际上有四种不同类型的Python组件，我们之后在讨论这个。我们将深入研究输入输出1:1的同步模块，因为用它来做解释很简单。
现在我们知道了使用python去写一个我们自己的模块，同时添加一个同步器模块到我们的模型中。我们再一次运行gr_modtool帮助指令，知道我们熟悉这些不同的命令。我们看到add就是我们要找的。现在我们运行add命令的帮助指令，为了查看我们需要输入的信息。

我们看到-l LANG同时-t BLOCK_TYPE跟我们的例子相关联。因此当我们创建一个新的模块时，我们需要知道这些指令。当提示输入一个名字时，输入“multiply_py_ff”,当提示输入参数时，输入“多个(multipe)”,当提示使用Python QA标准时，输入“y”，或者只是回车。

我们注意到有5个变化：两个变化针对于CMakeLists.txt文件，一个是用来测试我们代码的qa_multiply_py_ff.py，一个新文件关于功能部分的multiply_py_ff.py,和一个用来连接模块到GRC的文件tutorial_multiply_py_ff.xml,所有的这些都发生在python和grc文件夹中。

对于有严格类型的块，我们使用后缀来表明输入输出的类型。这个模块主要针对于浮点型，所以我们使用后缀_ff：浮点型输入浮点型输出。或者只有_f表示信宿或者信源使用的是浮点型。

3.2.5 XML文件
