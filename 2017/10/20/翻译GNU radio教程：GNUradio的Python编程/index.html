3.1介绍Python编写GNURadio程序

这个教程一共有3个部分。第一个部分是如何去修改，创造或者简单理解GRC产生的Python文件。第二部分是如何从头开始创造我们自定义的面向对象的模块。最后我们通过一个实际的工程去得到更多的练习，同时关于在我们自己的工程中建造一个大体印象。和上一个教程一样，所有的内容包括图片，源程序和grc文件，包含在github界面,我们可以按照GRC教程中的指导下载到本地。
同样我们需要建两个文件夹，一个是“解决方案”文件夹，一个是“工程”文件夹：

/home/user/gnuradio/tutorials/solutions
/home/user/gnuradio/tutorials/work

一般来说，如果我们将鼠标悬停在GRC流程图上，我们能够看到相应的文件名。同样也应用于其他图片上。

3.1.4 一起连接到模块

连接模块所使用的函数是self.connect(block1,block2,block3,…),这样他将会连接block1的输出与block2的输入，block2的输出和block3的输入等等。我们能够会用一个connect()连接多个模块，但这只对一对一的通信有效。如果我们回到我们初始化的流程图，Audio Sink模块有两个输入，连接他们的方法是使用上面的语句。这第一个连线只连接src0（350Hz 波形）到dst的第一个输入，第二个连线是将src1（440Hz波形）连接到dst的第二个输入。到目前为止代码跟我们在开始创建流程图的代码是一样的，其余的行只是启动流程图，并提供了一个键盘中断。

当我们不需要使用GRC创建Python文件，我们已经经过了学习GNU Radio的初级阶段。我们能够通过点击GRC的模块，而不是使用python去构建流程图。使用python则会让我们每次运行GRC有一个更清晰的理解，可以更好的控制程序去完成我们想让它做的。
